Empty Bundle Validation Bug

Build validation does not detect when the generated CSS bundle is empty.

Issue #5759

The Problem

When the build step fails silently and produces an empty easemotion.min.css, the validation script checks file existence but not content. Users deploying the build get zero styles with no warning.

Before vs After

Broken — No Content Check

Validates Existence Only

⚠️ easemotion.min.css (0 bytes) ✓ Passed

Bundle exists but is empty — validation passes incorrectly.

Fixed — Content Validation

Validates Content + Existence

✓ easemotion.min.css (48 KB) ✓ Passed
⚠️ easemotion.min.css (0 bytes) ✗ Failed

Bundle size is validated — empty bundles trigger a build failure.
